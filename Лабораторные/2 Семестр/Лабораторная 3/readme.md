# Лабораторная работа: Система управления библиотекой


## Особенности реализации

Использование терминала для взаимодействия с программой `(CTRL + ~ )`:

| Действие | Команда |
| :--- | :--- |
| **Добавить книгу** | `node lb.js add "Название" "Автор" "ISBN" Кол-во` |
| **Новый читатель** | `node lb.js adduser "Имя" "Фамилия" "ID_карты"` |
| **Выдать книгу** | `node lb.js issue "ISBN" "ID_карты"` |
| **Список фонда** | `node lb.js list` |

---

## Примеры выполнения

### 1. Добавление данных
```bash
node lb.js add "Гарри Поттер" "Роулинг" "555-133" 10
node lb.js adduser "Иван" "Иванов" "CARD-1"
```


### 2. Выдача книги (Операция Issue)

```Bash
node lb.js issue "555-133" "CARD-1"
```


### 3. Просмотр текущего состояния (List)
```Plaintext
--- КНИЖНЫЙ ФОНД ---
555-133 | "Гарри Поттер" | В наличии: 9
```
> `Почему число изменилось?` <br>Изначально было `10 штук`. После вызова команды `issue` один экземпляр закрепился за пользователем `"Иванов"`, поэтому в свободном доступе осталось `9`.
---

### Структура базы данных
Программа разделяет сущности по разным файлам для удобства администрирования:

`books.json` <br>
Содержит реестр книг и складской учет (инвентарь).

```JSON
{
  "list": [
    {
      "id": 1711452600,
      "title": "Гарри Поттер",
      "author": "Роулинг",
      "isbn": "555-133"
    }
  ],
  "inventory": {
    "555-133": 9
  }
}
```

`users.json` <br>
Содержит базу пользователей и список ISBN книг, которые находятся у них на руках.

```JSON
[
  {
    "fName": "Иван",
    "lName": "Иванов",
    "card": "CARD-1",
    "books": ["555-133"]
  }
]
```
